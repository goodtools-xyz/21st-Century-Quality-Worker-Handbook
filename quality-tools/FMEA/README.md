# FMEA (Failure Mode and Effects Analysis)

## 概述

FMEA（失效模式与影响分析）是一种系统化的可靠性分析方法，用于识别产品或过程中潜在的失效模式、评估其影响严重程度，并确定优先级以制定改进措施。FMEA作为一种预防性质量工具，广泛应用于产品设计和制造过程中，有助于在问题发生前识别和消除潜在风险。

## FMEA的类型

- **SFMEA (System FMEA)**: 系统FMEA，从系统层面分析失效模式和影响
- **DFMEA (Design FMEA)**: 设计FMEA，用于分析产品设计中潜在的失效模式及其影响
- **PFMEA (Process FMEA)**: 过程FMEA，用于分析制造和装配过程中潜在的失效模式及其影响
- **MFMEA (Machine FMEA)**: 设备FMEA，专注于设备相关的失效分析

## FMEA的实施步骤

1. **准备与计划**：确定范围、组建团队、收集信息
2. **功能分析**：明确定义产品/过程的功能和要求
3. **失效模式识别**：识别潜在的失效模式
4. **影响分析**：评估失效对系统/用户的影响
5. **原因分析**：确定可能导致失效的原因
6. **风险评估**：使用SOD评分确定风险优先级
7. **改进措施**：制定并实施改进建议
8. **跟踪与更新**：验证措施有效性并持续更新FMEA

### PFMEA（过程FMEA）

### FMEA-MSR（监视与系统响应）
**[查看详细内容](./FMEA-MSR/README.md)**

MSR是AIAG & VDA 2019标准中新增的重要组成部分，关注在设计或过程中实施的监视机制和系统响应策略，用于及时发现和应对潜在的失效或风险。

### AP表（行动优先级判定表）
**[查看详细内容](./AP表/README.md)**

AP表用于根据严重度（S）、频度（O）和探测度（D）的组合，确定失效风险的优先级。根据AIAG & VDA 2019标准，本页面提供统一的SOD矩阵行动优先级判定表。

## 最佳实践与注意事项

- 确保FMEA团队包含跨职能的专业人员
- 定期更新FMEA文档，特别是在设计或过程变更后
- 将FMEA与其他质量工具如控制计划、SPC等结合使用
- 关注高优先级风险项目的措施实施和验证
- 培训相关人员理解FMEA的重要性和正确使用方法
